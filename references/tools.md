# 可选工具

只在当前环境缺少网页/小红书读取能力，或用户明确询问如何安装时读取本文件。

## 推荐

- Agent-Reach：<https://github.com/Panniantong/Agent-Reach>
  - 用途：搜索和读取多个社交平台。
  - 注意：部分平台需要 Cookie 或登录状态。
- xiaohongshu-cli：<https://pypi.org/project/xiaohongshu-cli/>
  - 用途：小红书搜索、笔记读取等结构化操作。
  - 注意：可能需要登录 Cookie；平台接口和风控可能变化。
- Playwright：<https://playwright.dev/docs/intro>
  - 用途：当 Agent 环境允许时自建浏览器读取和 HTML 验证。

## 安装原则

- 优先使用宿主环境已有的浏览器/连接器，不重复安装。
- 先说明作用、需要的权限和风险，再征得用户同意。
- 不安装来源不明的插件，不索要用户密码，不输出 Cookie。

## 无法访问小红书时

按顺序判断并告诉用户具体缺哪一层：

1. 当前 Agent 是否有浏览器控制、社交连接器或小红书工具。
2. 工具是否能控制用户正在使用的本地浏览器，而不是隔离的无登录浏览器。
3. 用户是否已在该浏览器中自行登录并明确授权访问。
4. 页面是否因验证码、频控或反自动化机制无法继续。

推荐顺序：宿主已有浏览器能力 > 本地浏览器由用户手动登录并授权 > 本地连接器/CLI > 用户提供链接和截图。不要要求用户在对话中发送密码或完整 Cookie。

如果只能使用通用网页搜索，必须称为“外部搜索待验证”，不能声称已在小红书站内搜索，也不能输出未经证实的粉丝量、爆款数据或正式热度排名。

## 截图提交清单

当连接器不可用、模型拒绝访问或平台拦截时，请用户按优先级提交：

1. 主页完整截图：头像、账号名、简介、粉丝及主页可见互动数据。
2. 笔记列表截图：尽量覆盖最近 20-30 篇，能看清标题、封面和互动数。
3. TOP 5 笔记详情：标题、发布日期、点赞、收藏、评论及正文。
4. 3-5 张代表封面原图或完整截图。

材料不全也可以开始，但必须标注缺失项；不要反复要求用户重新提交已经清晰可见的内容。
