# ache chaijie skill

一个用于小红书账号和竞品拆解的可移植 AI Skill。

它不只总结账号，还会分析内容定位、公开笔记热度、爆款公式、真实封面、可复制点和下一步内容方向。

## 功能

# 小红书账号拆解专家

一句话拆透一个账号。

输入 1-5 个小红书账号链接，自动生成可视化分析报告，拆解账号定位、内容模型、爆款逻辑、增长路径和可复制策略，帮助快速找到值得学习的对标对象。

## 核心能力

### 账号深度拆解

基于真实主页和公开内容，分析：

* 账号定位与人设
* 内容方向与选题策略
* 爆款内容模型
* 封面与标题规律
* 评论区运营逻辑
* 商业化路径
* 内容迭代过程
* 当前增长策略

重点回答：这个账号为什么能做起来。

### 智能寻找对标账号

如果用户没有现成对标账号，会根据赛道、内容类型、目标用户和转化目标自动推荐：

* 2 个成熟标杆账号
* 3 个当前可对标账号

明确区分：

* 已验证的小红书主页账号
* 外部搜索待验证账号

避免错误对标。

### 爆款模式识别

识别账号内容中的：

* 有效复用
* 模板迭代
* 内容矩阵
* 机械复制

判断账号的增长来自偶发爆款，还是已经形成稳定的内容系统。

### 封面与内容结构分析

直接分析真实笔记样本，包括：

* 封面文案
* 主体设计
* 构图方式
* 配色风格
* 视觉钩子
* 信任证据
* 内容结构
* 评论反馈

避免只看数据，不看内容。

### 账号成长路径还原

当公开样本充足时，分析：

* 早期探索阶段
* 内容转折点
* 爆发阶段
* 当前稳定打法

帮助理解账号是如何一步步跑出来的。

## 分析原则

所有结论仅基于公开可验证信息：

* 主页信息
* 真实笔记内容
* 封面素材
* 标题结构
* 评论互动
* 公开可见数据

不会编造：

* 阅读量
* 流量数据
* 转化数据
* 商业合作数据
* 任何后台指标

无法确认的信息会明确标注。

## 输出内容

默认生成单页 HTML 可视化报告，包含：

* 账号画像
* 内容模型分析
* 爆款案例拆解
* 成长路径复盘
* 对标学习建议
* 三阶段行动路线图

报告中的账号、笔记、案例和封面均保留原始链接，可直接访问对应内容。

支持：

* 单账号深度拆解
* 2-5 个账号横向对比
* 对标账号发现
* 爆款模式分析

## 输出结果

最终不仅告诉你账号做了什么，更会回答：

* 为什么能火
* 什么值得学
* 新号先学什么
* 如何验证打法
* 什么时候扩展
* 下一步应该怎么做

目标是把优秀账号的增长逻辑，拆解成可以直接执行的动作。

## 使用方式

### 直接分析账号

```text
请分析这个小红书账号：
https://www.xiaohongshu.com/user/profile/EXAMPLE
```

Skill 会直接开始，不强制用户先回答定位问题。

分析完成后会在当前工作区写入可直接打开的 HTML 报告，聊天中只保留关键结论和报告链接。

### 先找对标账号

```text
我想做一个职场效率账号，请帮我找适合的小红书对标账号。
```

Skill 第一轮会邀请用户直接提交已有对标链接；没有现成账号时，再问一个高信息量问题并用选择题补齐定位。用户确认账号画像后，才会搜索并给出 5 个候选账号。

## 安装

### Codex 或支持 SKILL.md 的 Agent

1. 下载或 clone 本仓库。
2. 将整个 `ache-chaijie-skill` 目录放入 Agent 的 skills 目录。
3. 确保入口文件名为 `SKILL.md`。

WorkBuddy 或其他 Agent 如果支持从 GitHub 导入 Skill，可直接导入本仓库；如果只支持指令文件，加载 `SKILL.md` 即可。具体安装入口以宿主产品文档为准。

## 可选工具

Skill 不强制依赖某一个插件。如果宿主 Agent 缺少小红书或浏览器读取能力，可考虑：

- [Agent-Reach](https://github.com/Panniantong/Agent-Reach)
- [xiaohongshu-cli](https://pypi.org/project/xiaohongshu-cli/)
- [Playwright](https://playwright.dev/docs/intro)

平台可能需要登录或 Cookie。优先让用户在本地浏览器中自行登录并授权 Agent 访问，不要把密码或 Cookie 粘贴到对话、仓库或公开文件中，也不要绕过验证码、权限或平台风控。

如果工具不可用、模型拒绝访问或平台拦截，Skill 会先检查并询问宿主 App 的内置浏览器、Skills、连接器和本地自动化方案。确认内置路径都不可用后，才推荐外部连接器或截图提交方式，不会把第一次“无法访问”当作最终结果。

## 数据与隐私

- 只分析公开可见内容或用户主动提供的数据。
- 无法获取的数据会标注“未公开/未获取”。
- 示例使用虚构用户设定，不包含创建者的公司、履历或私人账号信息。
- 小红书网页结构和反自动化规则可能变化，实际可用数据以运行时为准。
